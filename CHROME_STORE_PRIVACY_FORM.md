# Chrome 商店隐私权表单填写指南

## 📋 表单填写内容

### 1️⃣ 单一用途说明（最多 1000 字符）

```
GrassTab 是一个美观实用的新标签页扩展程序，其单一用途是为用户提供可自定义的浏览器起始页体验。

核心功能包括：
• 管理和访问常用网站快捷方式
• 展示实用小组件（时钟、天气、日历等）
• 自定义壁纸和界面设置
• 提供便签和计算器等工具
• 可选的 AI 助手功能（需用户自行配置）

所有功能都围绕"提供一个功能丰富的个性化新标签页"这一单一目的。用户可以根据自己的需求选择启用或禁用各项功能，所有数据均存储在本地浏览器中，不会上传到任何服务器。
```

### 2️⃣ 需请求 storage 的理由（最多 1000 字符）

```
storage 权限用于在用户的浏览器本地存储以下数据：

1. 应用快捷方式配置 - 用户添加的网站快捷方式、图标和布局
2. 系统设置 - 用户的个性化设置（语言、主题、显示选项等）
3. 壁纸选择 - 用户选择或上传的背景图片
4. Dock 栏配置 - 底部快速访问栏的应用排列
5. 便签内容 - 用户记录的备忘录和待办事项
6. 小组件设置 - 时钟、天气、日历等小组件的配置
7. AI API 配置 - 用户选择使用 AI 功能时配置的 API 密钥（仅本地存储）

所有数据仅存储在用户的浏览器本地，不会上传到任何远程服务器。这是扩展程序实现其核心功能（提供个性化新标签页）所必需的权限。
```

### 3️⃣ 需请求 unlimitedStorage 的理由（最多 1000 字符）

```
unlimitedStorage 权限用于突破 Chrome 默认的 5MB 存储限制，允许用户存储更多内容：

1. 高分辨率壁纸 - 用户可能上传高质量的自定义背景图片（单张可能超过 2MB）
2. 多个壁纸 - 用户可以保存多张壁纸供切换使用
3. 大量应用快捷方式 - 支持用户添加数十甚至上百个常用网站
4. 自定义图标 - 用户为应用上传的自定义图标（base64 格式）
5. 代码编辑器内容 - Monaco Editor 中用户保存的代码片段
6. 便签内容 - 用户的长篇笔记和待办事项
7. 配置备份 - 完整的配置数据可能随着使用时间增长而增大

由于新标签页是用户频繁访问的页面，我们希望为用户提供足够的存储空间来保存他们的个性化设置和内容，确保良好的使用体验。所有数据仅存储在本地，不会传输到服务器。
```

### 4️⃣ 需请求主机权限的理由（最多 1000 字符）

```
主机权限 (https://*/*) 用于实现以下功能，不会收集或上传用户的个人数据：

1. 获取网站图标（Favicon）
   - 当用户添加网站快捷方式时，自动获取该网站的图标
   - 调用公开的图标服务 API（icon.horse, clearbit.com 等）
   - 仅发送网站域名，不发送用户浏览历史或个人信息

2. 搜索建议功能（可选）
   - 用户在搜索框输入时提供搜索建议
   - 调用搜索引擎的公开 API（Google Suggest, Bing 等）
   - 用户可在设置中完全禁用此功能

3. 嵌入式网页小组件
   - 允许用户在小组件中嵌入网页内容（如天气预报、新闻等）
   - 仅在用户明确添加嵌入网页小组件时使用
   - 用户完全控制嵌入哪些网页

所有网络请求都是为了提供用户明确要求的功能，不会在后台收集或传输任何用户数据。
```

### 5️⃣ 您正在使用远程代码吗？

**选择**: ❌ **不，我并未使用远程代码**

（不需要填写理由，因为我们确实没有使用远程代码，所有 JavaScript 都打包在扩展程序中）

## 💡 重要提示

1. **保持一致性** - 表单中填写的内容要与隐私政策页面内容一致
2. **诚实透明** - 准确描述扩展的功能和权限用途
3. **详细说明** - 不要简单写"用于存储数据"，而要具体说明存储什么数据
4. **强调本地存储** - 多次强调所有数据仅本地存储，不上传服务器
5. **用户控制** - 说明用户可以控制和删除所有数据

## 📞 如果被拒绝怎么办？

1. 仔细阅读拒绝邮件中的说明
2. 根据反馈修改相应内容
3. 更新表单或代码
4. 重新提交审核

常见拒绝原因：
- 权限说明不够详细 → 补充更具体的使用场景
- 隐私政策链接失效 → 确保页面可访问
- 功能描述不清 → 使用更简单明确的语言
- 数据收集声明不一致 → 确保表单和政策一致
